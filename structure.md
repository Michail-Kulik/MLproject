Вот полный код для наглядности:

python
class LSTMForecaster(nn.Module):
    def __init__(self, n_features: int, hidden_size: int = 64, 
                 num_layers: int = 2, dropout: float = 0.1, horizon: int = 14):
        super().__init__()
        self.horizon = horizon

        self.lstm = nn.LSTM(
            input_size=n_features,
            hidden_size=hidden_size,
            num_layers=num_layers,
            dropout=dropout if num_layers > 1 else 0.0,
            batch_first=True,
        )
        self.head = nn.Sequential(
            nn.LayerNorm(hidden_size),
            nn.Linear(hidden_size, horizon),
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        out, _ = self.lstm(x)        # (B,T,H)
        last = out[:, -1, :]         # (B,H)
        return self.head(last)       # (B,horizon)
1. Инициализация (__init__) — сборка скелета
n_features: int
Сколько колонок признаков подается на вход на каждый день. В вашем проекте это len(FEATURE_COLS) (например, sales, dayofweek, is_holiday и т.д.).

hidden_size: int = 64
Размерность скрытого состояния (векторов h и c). Это «объем памяти» LSTM. 64 — значит, в каждый момент времени сеть помнит 64 числа о прошлом.

num_layers: int = 2
Количество слоев LSTM, поставленных друг над другом. Два слоя позволяют выучивать более сложные временные зависимости (первый слой ловит локальные паттерны недели, второй — сезонность месяца).

dropout: float = 0.1
Вероятность зануления нейронов между слоями для борьбы с переобучением.
Важный нюанс: Строка dropout=dropout if num_layers > 1 else 0.0.

Если слой один (num_layers=1), то dropout обязан быть 0.0, иначе PyTorch выбросит ошибку, потому что дропаут применяется между слоями, а если слой один — между чем и чем?

horizon: int = 14
Сколько дней вперед нужно предсказать.

batch_first=True
Критическая настройка. Без нее входной тензор ожидается в формате (Time, Batch, Features). С ней — (Batch, Time, Features). Так удобнее работать с DataLoader и батчами, потому что первая размерность всегда батч.

2. Слой self.lstm — мозг модели
python
self.lstm = nn.LSTM(
    input_size=n_features,  # Что приходит каждый день (например, 10 фич)
    hidden_size=hidden_size, # Размер памяти (64)
    num_layers=num_layers,  # Этажерка из 2-х слоев
    dropout=dropout,
    batch_first=True,
)
На этом этапе создается вычислительный граф, который умеет:

Принять батч размером (B, T, n_features)

Выдать на выходе последовательность (B, T, hidden_size)

Внутри хранить и обновлять состояния (h, c).

3. Слой self.head — "голова" для предсказания
python
self.head = nn.Sequential(
    nn.LayerNorm(hidden_size),
    nn.Linear(hidden_size, horizon),
)
Первый элемент: nn.LayerNorm(hidden_size)

Зачем: LSTM имеет свойство выдавать значения, которые "гуляют" по диапазону после долгих вычислений. Нормализация приводит вектор размером 64 к единому масштабу (среднее 0, дисперсия 1).

Результат: Делает обучение более стабильным и быстрым, особенно когда hidden_size соединяется с линейным слоем.

Второй элемент: nn.Linear(hidden_size, horizon)

Матричное умножение: превращает вектор [64] в вектор [14].

Каждое из 14 чисел на выходе — это прогноз продаж на конкретный день в будущем (t+1, t+2, ..., t+14).

4. Метод forward — рабочий цикл
Когда вы вызываете model(x), происходит следующая цепочка действий:

Шаг 1: Прогон через LSTM
python
out, _ = self.lstm(x)
Вход x: Тензор размером [Batch, 28, n_features] (для вашего примера).

Выход out: Тензор [Batch, 28, 64].

Что в _? Это кортеж (h_n, c_n) — финальные состояния памяти после обработки всего батча. Мы их игнорируем, потому что используем все промежуточные выходы out.

Шаг 2: Извлечение последнего временного шага
python
last = out[:, -1, :]  # [Batch, 64]
out[:, -1, :] — синтаксис NumPy/PyTorch.

: — взять все батчи.

-1 — взять последний индекс по оси времени (день 28).

: — взять все 64 признака скрытого состояния.

Смысл: Мы считаем, что в последнем скрытом состоянии уже содержится вся информация о предыдущих 27 днях, сжатая через механизмы памяти LSTM. Именно его мы подаем на "голову" для прогноза.

Шаг 3: Финальное преобразование
python
return self.head(last)  # [Batch, 14]
Сначала LayerNorm выравнивает last.

Затем Linear превращает 64 числа в 14 чисел прогноза.

Резюмируем в виде таблицы размерностей
Этап	Описание	Размерность тензора
Вход модели	x	[B, 28, F] (F = кол-во фич)
После LSTM	out	[B, 28, 64]
Взятие среза	last	[B, 64]
После LayerNorm	normalized	[B, 64]
После Linear	Выход модели	[B, 14]
Почему это работает для прогнозирования?
Стандартный LSTM может выдавать последовательность ответов на каждый день (many-to-many). Но ваша задача — спрогнозировать сразу окно из 14 дней, имея на входе только последние 28 дней.

Используя схему "Весь вход -> Последнее скрытое состояние -> Линейный слой", вы заставляете нейросеть упаковать всю информацию о прошлом в один вектор размером 64, а затем распаковать этот вектор в картинку будущего на 14 дней. Это классическая архитектура Sequence-to-Vector, идеально подходящая для вашей задачи.
